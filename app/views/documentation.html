The prototype of the Search Guardian, a privacy web application, was developed for the purpose of this study as one of its requirements. This application aims to protect the privacy of its users or at least to improve their well-being by showing them information concerning them available on the Web and by giving them a chance to handle their personal information according to their desires.

The web application was mainly developed using Yeoman (Grunt and Karma), AngularJS and Karma. This is only a prototype of the SearchGuardian as the Web is so vast it was impossible to cover it all in such short terms. The web application is still in development and the last version of the project is available at this URL: https://github.com/jtran1403/SearchGuardian

To use the features of the applications, you will have to log in your Facebook accounts or fake ones you created in order to retrieve the data. Facebook features

Log in your Facebook account and access most of your personal information (see documentation of Graph API).

The privacy value and a link to the albums associated with your profile are available in the table. With this, it is possible to go straight to the album and to change the privacy value if you wish to. Clicking on the button in the first column of the table will open a scroll pane below where you will be able to view the pictures in the album you selected.

Your last activities (e.g. commenting a picture and posting on a wall) on Facebook are recorded in your activity log. The application returns part of the activity log and allow you to check who can see them with the privacy value returned. You will notice some privacy parameter are returned as "", that is because graphAPI returned no value which means the privacy setting of this post depends on the privacy of the wall you published it. For instance, if you posted a comment on the wall of one of your friend and the privacy of his wall is set to viewable by "Friends", then the privacy of your comment is set to the friends of the wall's owner as well. On the other hand, if the privacy is set to "Everyone", then anybody can see that comment. In that case, even Internet users not logged on Facebook can access your post if they possess the URL which is also provided by the application.

LinkedIn features

LinkedInAPI only allows us to read information and offers a very limited freedom to its consumers. Through the application, the user can only log in his/her LinkedIn account and review the information he entrusted.

EXIF data manipulation

The page accessed by the button entitled "Geolocation&pictures" enables the user to view the EXIF data registered on the picture if the picture is in the format.jpeg, .gif and .png and if its size is smaller than 3Mo only. By changing the value of the dropdown list below the drag and drop zone, the user can also generate this very same picture but without the GPS data contained in the EXIF data. This is only possible if the picture is smaller than 1.5Mo though because the burden from generating and encoding the data for the picture is too heavy for the browser.

Future search engine feature

A Google search engine allows the user to find information about someone on the websites I associated to the search engine. The development of this feature is not over yet but will provide search results pertinent to the profile associated above the search engine.

Expected software failure:
  • Deactivated Javascript on the browser used
  • Scripts blocked by plugins installed on the browser
  • Loading failures for various reasons (e.g. already logged in the network targeted, previous uses of the application on the machine...)
Respective solutions:
  • Activate Javascript via the options or settings of your browser
  • Deactivate plugins blocking the scripts
  • Refresh the page
Pertinent comments
  • The Graph API from the Facebook features requires the user to change the URL if it contains the IP address instead of 'localhost' (e.g. '127.0.0.1:9000/' becomes 'localhost:9000/'), else it does not even load the API.
  • The APIs used by the application will ask the permission to access information available in your Facebook account for instance; just log in and accept the conditions. This occurs the first time the user tries out the application on his/her machine or if the browser does not memorize this information.
  • Most of the personal information available through the API is displayed which does not mean all information entrusted to the service provider is shown.
  • Since you gave permission to the application to access that data, it will retrieve it no matter what the privacy setting is. Besides, the privacy settings of the profile cannot be returned by using Graph API, therefore if you never set it properly, do check them and change them if reguired.
  • You can test the privacy of your comment by opening a different web browser where you are not logged in Facebook and pasting the URL in the browser. You will see the post even though you are not logged in Facebook.
